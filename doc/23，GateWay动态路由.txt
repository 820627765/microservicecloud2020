1，没有网关之前，微服务访问其他微服务，使用 Ribbon 来做负载均衡（客户端负载均衡）。
2，现在有了网关之后，我们就需要网关来做同一个微服务集群的负载均衡了，怎么做呢？
   默认情况下 Gateway 会根据注册中心注册的服务列表，以注册中心上的“微服务名”为“路径”创建
   “动态路由”进行转发，从而实现动态路由的功能。

   那么：如何让Gateway 实现动态路由的功能呢？
   1，修改application.yml ，使得gateway开启从注册中心动态创建路由的功能，利用微服务名进行路由
      pring:
        cloud:
          gateway:
            discovery:
              locator:
                enabled: true # 开启从注册中心动态创建路由的功能，利用微服务名称进行路由
   2，将yml配置的路由 或 代码配置的路由 的 uri，修改为使用“服务名”来进行配置，如：
      uri: lb://微服务名


   #如此，当通过网关访问地址，当匹配某个路由后，gateway 将通过配置的uri的微服务名去服务注册中心获取
   #对应的服务列表，然后默认按照轮询的方式进行转发。
